Exercise 3: Programmatically end an Activity

The goal of this exercise is to demonstrate how to end an Activity programmatically. This is useful whenever you need to implement something like a Save or Cancel button in your user interface.

You will write code to end the AddItemActivity when the user clicks the Cancel button. The image below shows a screenshot of every Activity in the app - the arrow depicted in red shows the Activity transition that you will code in this exercise.

To complete the exercise, you will need Visual Studio for Windows or macOS with the Xamarin development tools installed. You will also need either an emulator/simulator or a device to run the exercise on. Please see the setup page if you need help installing the Xamarin development environment.

Finish an Activity

Your goal is to programmatically end the AddItemActivity when the user clicks on the Cancel button. The button and a stub click handler are already written for you - your job is to complete the implementation of the handler.

  1. Open AddItemActivity.cs.
  2. Locate the OnCancelClick method.
  3. Inside OnCancelClick, add a call to the Activity's Finish method.
void OnCancelClick(object sender, EventArgs e)
{
   Finish();
}
  1. Run the application to test your work.
  2. Navigate to the AddItemActivity.
  1. Click the Cancel button and verify you return to the MainActivity.

Exercise summary

In this exercise, you programmatically ended an Activity.

This is a nice technique to know; however, you should rarely need it since the standard back-button should work most of the time. The motivating case we looked at is a classic example: a Cancel button that returns to the previous Activity without saving the user's data.
